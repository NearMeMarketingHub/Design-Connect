The plan must answer these business-rule questions before any build happens:

1. Which settings should affect estimator totals?
2. Should overhead apply to subtotal only, or subtotal plus category adjustments?
3. Should markup/profit apply to subtotal only, subtotal plus overhead, or final cost after adjustments?
4. Should labor burden apply only to labor line items?
5. Should material markup apply only to material line items?
6. Should subcontractor markup apply only to subcontractor line items?
7. Should equipment/operating cost percentage be a markup, an added overhead line, or only a reference field?
8. How do we avoid double-counting price book items that already include burdens, material fees, labor rates, subcontractor rates, or retail prices?
9. Should these settings be automatic, optional toggles, or prefilled editable fields?
10. What happens when settings are null?
11. What happens when settings change after an estimate or budget already exists?
12. Should saved estimates store a snapshot of the percentages used at save time?
13. Should project budgets store a snapshot of the percentages used at creation time?
14. Which old records should remain unchanged?
15. What should be shown to users so they understand these are defaults, not guaranteed final margins?

Preferred direction unless there is a strong reason otherwise:

* Settings should apply to new work only.
* Existing estimates, invoices, budgets, expenses, and projects must not change.
* Defaults should prefill editable fields before they automatically affect totals.
* Any automatic adjustment should be visible as a named line item or named summary line.
* Users should be able to override applied percentages per estimate/project.
* Saved estimates should probably snapshot the applied percentages so future company setting changes do not rewrite history.
* Project budgets created from estimates should probably inherit the estimate snapshot.
* Empty/manual budgets should only apply company defaults if the user explicitly enables them.
* Price book items need special care because their retail price may already include markup/burden.
