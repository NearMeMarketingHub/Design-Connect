Now we are moving into selections / job book functionality.

Goal:
Add a Selections / Job Book foundation so contractors can create, organize, track, and optionally expose project selections to clients.

Selections are project-specific choices such as:

* Flooring
* Cabinets
* Countertops
* Paint colors
* Fixtures
* Appliances
* Tile
* Doors/windows
* Hardware
* Lighting
* Plumbing fixtures
* Other finish/material selections

Primary contractor story:
A company owner/admin or permitted internal team member opens a project and creates selection items grouped by category or room. Each selection can track requested option, selected option, vendor, allowance/budget amount, actual cost if known, status, due date, notes, files/links, and whether it is visible to the client.

Primary client story:
A client can view client-visible selections, see what is pending/approved, and possibly provide a choice or approval if that is safe for this first version.

Preferred direction unless the code suggests otherwise:

* Add a dedicated `project_selections` table.
* Keep selections project-scoped and company-scoped.
* First version should prioritize manual create/edit/delete and clear tracking.
* Add a Project Detail tab called “Selections” or “Job Book.”
* Company Owner and Company Admin can create/edit/delete selections.
* Regular internal contractors can view; editing can be allowed only if existing app permissions already support similar internal editing safely.
* Clients should only see selections where `clientVisible = true`.
* Client response/approval can be planned, but keep it simple for first version.
* Do not connect selections to budgets, expenses, invoices, or financial totals yet.
* Do not add automatic budget changes based on selections.
* Do not add purchase orders yet.
* Do not add notifications/emails yet.
* Do not add permitting, production sheets, daily logs, or payouts.
* File/image upload should be included only if the app already has a safe upload pattern; otherwise defer.
* Product/vendor URL fields are safe to include now.
* Import can be deferred unless there is a very simple low-risk CSV/copy-paste approach.

Possible first version:

* New Selections tab on project detail page.
* Selection list grouped by category/status.
* Create/edit/delete dialog.
* Fields: category, room/area, title, description, status, dueDate, selectedOptionName, selectedOptionDetails, vendorName, productUrl, allowanceAmount, estimatedCost, actualCost, clientVisible, notes/internalNotes.
* Simple status badges.
* Optional summary cards: Pending, Client Review, Approved, Ordered/Received.
* Client-visible flag stored now.
* Client portal read-only display can be planned but not overbuilt unless there is already a safe place to add it.
