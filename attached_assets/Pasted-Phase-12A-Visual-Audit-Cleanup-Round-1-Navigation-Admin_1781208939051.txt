2. Fix Admin/Audit Log raw database error.

Admin pages are showing large raw database/code error blocks. One observed error:

“op ANY/ALL (array) requires array on the right side”

Investigate likely affected backend query code, especially Audit Log/company enrichment logic.

Likely fix:
Replace unsafe raw ANY/ALL array SQL usage with Drizzle-safe `inArray(...)` queries or otherwise ensure the right side is a real SQL array.

Search for patterns like:

* `ANY(`
* `ALL(`
* raw SQL using arrays
* audit log company enrichment queries

Do not change intended filtering behavior.

3. Improve admin error display.

Admin pages should not show giant raw database/code blocks to users.

If an admin query fails:

* show a clean destructive Alert/card/toast
* use parseErrorMessage where appropriate
* keep detailed raw error logging in console/server logs only

Do not swallow errors silently.
