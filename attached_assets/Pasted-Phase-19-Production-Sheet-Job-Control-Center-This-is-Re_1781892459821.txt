Goal:
Add a project-level Production Sheet / Job Control Center foundation that gives contractors one operational overview of the job.

This should help company owners/admins and internal contractors quickly see and manage the key job-control details for a project:

* project identity/details
* client/contact info
* project status
* schedule/timeline highlights
* permitting highlights
* selections highlights
* budget/expense summary
* important internal notes
* production/job-control checklist items
* key job dates
* internal job readiness/status indicators

Important:
This phase should NOT guess payout, commission, Near Me 1%, revenue-recognition, or payment formulas.

Those need separate approval later.

Preferred direction unless the code suggests otherwise:

* Add a new Project Detail tab called “Production” or “Job Control.”
* Keep this project-scoped and company-scoped.
* Use a one-to-one production sheet record per project if editable production-specific fields are needed.
* Use existing data for read-only summaries:

  * budget totals
  * expense actuals
  * timeline item counts/status
  * permit status/blocking count
  * selections pending/approved counts
* Do not duplicate data already stored in timeline, permits, selections, budgets, or expenses.
* Company Owner and Company Admin can edit.
* Regular internal contractors can view; editing can be recommended only if it matches existing safe patterns.
* Clients should not see internal production sheet data in this phase.
* Subcontractors/notaries should not see production sheet data in this phase.
* Do not auto-update project status/progress from production sheet yet.
* Do not create financial formulas.
* Do not calculate payout, commission, Near Me 1%, margin, or profitability in this phase.
* Do not connect to Stripe/billing.
* Do not change invoices.
* Do not change estimate, budget, expense, selection, permit, or timeline behavior.
* Do not add notifications/emails yet.
