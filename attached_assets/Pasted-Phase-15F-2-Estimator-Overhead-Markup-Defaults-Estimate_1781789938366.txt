Phase 15F-2 — Estimator Overhead/Markup Defaults + Estimate Snapshot

Important:
Keep this as ONE single Replit task/build.

Do not split this into multiple smaller Replit plans.
Do not create separate follow-up task plans for the later Phase 15F items.
We will build future smaller steps separately later.

Use the prior Phase 15F planning work as background, but build only the scoped Phase 15F-2 work below.

Goal:
Replace the Estimator Calculator’s invisible hardcoded 10% overhead and 15% profit logic with visible, editable per-estimate Overhead % and Markup % fields.

These fields should prefill from company financial settings when configured. If company settings are blank, preserve existing behavior by using platform fallback values.

Approved business rules for this build:

1. Overhead formula:
   `overhead = subtotal * (overheadPct / 100)`

2. Markup/profit formula:
   `markup = (subtotal + overhead) * (markupPct / 100)`

3. Grand total formula:
   `grandTotal = subtotal + overhead + markup`

4. Null company settings fallback:
   If `defaultOverheadPct` is null, prefill Overhead % with platform fallback `10`.
   If `defaultMarkupPct` is null, prefill Markup % with platform fallback `15`.

Do not use 0% as the fallback.

5. Labels:

* If value comes from company financial settings, label it as “Company default.”
* If value comes from platform fallback, label it as “Platform default.”
* If user edits it, label it as “Custom.”

6. Percentage format:
   All percentages are whole-number percentages.
   Examples:

* 10 means 10%
* 15 means 15%
* 25.5 means 25.5%

At calculation time, use `value / 100`.

7. Existing records:
   Do not retroactively change existing estimates, budgets, invoices, expenses, projects, or price book items.
