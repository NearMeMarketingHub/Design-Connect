Phase 12D Patch 1: Estimator Calculator Pre-Project Quote Audit & Plan

Context:
The Estimator Calculator is intended to be a contractor-facing sales/quote tool used during meetings with potential clients.

Important product clarification:
The Estimator Calculator should connect to the company’s Price Book, which is set up and managed by the Company Owner or Company Admin. A contractor should be able to meet with a potential client, build an estimate/quote from that company Price Book, and save/export the result as a PDF so it can be printed or saved digitally.

This estimate may be created before a project exists.

Current issue:
There are currently two different estimator-related experiences:

1. `/contractor/calculator`

* sidebar-accessible
* connects to price book categories/items
* better matches the “contractor quickly builds quote from price book” concept
* but appears incomplete because it lacks full save/export quote workflow

2. `/company/estimates`

* accessible from Financials
* has project selector, manual line items, Save Estimate, Financial Summary, Recent Estimates
* but requires selecting an existing project, which does not fit pre-project client quoting

Goal of this audit:
Do not build the final estimator yet. First audit the current estimator/calculator data model, routes, UI, and PDF/export options so we can safely build one proper Estimator Calculator workflow next.

The final Estimator Calculator should:

* stay in the contractor/company sidebar for quick access
* connect to the company Price Book
* allow contractors to add price book items to an estimate/quote
* allow manual line items if appropriate
* allow prospect/client info before a project exists
* not require an existing project
* save the quote/estimate as a draft
* export/save the quote as a PDF
* support printing or digital saving
* optionally connect to a project later if the quote becomes a real job
* preserve existing project-based estimates where applicable

Out of scope for this audit:

* Do not build the final estimator yet
* Do not delete either estimator page yet
* Do not redirect routes yet
* Do not change schema yet
* Do not db:push
* Do not add Selections, expenses, reports, contractor sandbox, subcontractor switching, Stripe changes, or 3D changes

5. Define final desired workflow

Document this intended workflow:

* Company Owner/Admin sets up Price Book in Budget Manager
* Contractor opens Estimator Calculator from sidebar
* Contractor enters potential client/prospect details
* Contractor optionally selects an existing project, but this is not required
* Contractor builds quote using company Price Book items and/or manual line items
* Contractor reviews subtotal, labor/material, overhead/profit if supported
* Contractor saves the estimate as draft
* Contractor exports/saves the estimate as a PDF
* Contractor can print the PDF or send/save it digitally
* Later, accepted estimate can be connected to a project and possibly used for project budget creation
