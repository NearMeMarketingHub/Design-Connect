Phase 16 — Project Timeline / Gantt / Board View

Important:
This is Plan Mode only. Do not build yet.

Keep this as ONE single Replit plan. Do not split this into multiple separate Replit task plans.

Context:
Phase 15 is complete. The app now has a strong financial foundation:

* Estimates
* Estimate-to-project workflow
* Project budgets
* Project expenses
* Expense-to-budget actuals
* Company financial settings
* Estimator overhead/markup/category adjustment snapshots

Now we are moving into the project-management side.

Goal:
Add a project timeline / Gantt / board-view foundation so contractors can plan, track, and manage work phases/tasks inside each project.

This should become the backbone for later features like:

* Client portal progress visibility
* Selections / Job Book
* Permitting
* Production sheets
* Daily logs
* Notifications

Primary user story:
A company owner/admin or internal contractor opens a project and can create timeline items such as phases, tasks, or milestones with start/end dates, status, progress, and notes. They can view those items in a timeline/Gantt-style view and optionally a board/status view.

Client-facing visibility should be planned carefully, but do not overbuild the final client portal in this phase.

* Existing UI components/tabs/cards/tables
* Existing date formatting utilities
* Existing drag/drop or board UI patterns, if any
* Existing project chat/documents/progress components, if any

Preferred direction unless the code suggests otherwise:

* Add a dedicated project timeline items table.
* Keep project timeline company-scoped and project-scoped.
* Company Owner and Company Admin can create/edit/delete.
* Regular internal contractors can view; editing can be allowed only if existing app permissions already support internal contractor editing safely, otherwise keep edit to owner/admin.
* Clients should not see internal-only timeline data.
* Add `clientVisible` now if useful, but client-facing display can be deferred to Phase 21 unless there is already a safe client project page to show read-only visible items.
* Keep subcontractor/notary timeline access deferred.
* Do not auto-change project status or progress from timeline items yet.
* Do not connect timeline to budgets, expenses, invoices, permits, selections, daily logs, or notifications yet.
* Keep dependencies simple or defer them.
* Avoid complicated drag-and-drop unless the app already has a safe pattern for it.
* First version should prioritize clean data model, CRUD, and a useful project tab.

Possible first version:

* New Project Timeline tab on project detail page.
* Timeline item list/table grouped by status or date.
* Basic Gantt-style horizontal date visualization if straightforward.
* Optional board/status grouping if low-risk.
* Create/edit/delete dialog for timeline items.
* Status, dates, progress percent, item type, clientVisible flag.
* No auto-progress calculations.
* No financial changes.
* No client portal final buildout.
