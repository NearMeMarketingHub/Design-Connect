Phase 12F: Estimator Calculator Tabs + Saved Estimates

Context:
Phase 12E made `/contractor/calculator` the working sidebar Estimator Calculator for pre-project quotes using the company Price Book. It can now save estimates and export PDFs.

However, saved estimates are currently only visible through the Financials area / `/company/estimates`, which is confusing. The Estimator Calculator should be the main home for both creating estimates and viewing saved estimates.

Goal:
Update the sidebar Estimator Calculator page so it has two tabs:

1. Calculator
2. Estimates

The Calculator tab should keep the current Estimator Calculator workflow.

The Estimates tab should show saved estimates and allow basic estimate status management.

After this is done, remove the Estimator/Estimates access card from Financials so Financials is not the main place users go to manage estimates.

Out of scope:

* No schema changes unless absolutely unavoidable
* No db:push unless a schema change is unavoidable
* No company branding / logo / color customization yet
* No branded PDF redesign yet
* No project conversion workflow
* No create-project-from-estimate workflow
* No estimate approval signatures
* No Selections build
* No contractor sandbox
* No subcontractor multi-company switching
* No Stripe changes
* No 3D Floor Plan changes
* No expenses / reports / production sheets

1. Add tabs to `/contractor/calculator`

File:
`client/src/pages/contractor-calculator.tsx`

Add a tab layout near the top of the page, similar to the Admin Center tab style.

Tabs:

* `Calculator`
* `Estimates`

`Calculator`

The Calculator tab should contain the current estimator calculator UI:

* Price Book categories
* searchable item table

* Save Estimate
* Export PDF

2. Add Estimates tab

Inside the new `Estimates` tab, show saved estimates for the current company/user.

`GET /api/estimates`

The list should show:

* estimate custom ID
* client/prospect name
* job description / projectName
* date
* amount
* status
* actions

Suggested columns/cards:

* Estimate #
* Client / Prospect
* Job Description
* Date
* Amount
* Status
* Actions

Keep the layout clean and easy to scan.

3. Add basic status management

The Estimates tab should allow authorized internal company users to update an estimate status.

Supported statuses:

* `draft`
* `sent`
* `approved`
* `declined`

User-facing labels:

* Draft
* Sent
* Approved
* Declined

If an update estimate status API already exists, use it.

If not, add a small backend route for updating estimate status only.

Requirements:

* Status update must be company-scoped
* User must be an internal company user
* Do not allow clients, subcontractors, notaries, or cross-company users to update internal estimate statuses
* Do not allow users to update estimates from another company
* Do not change other estimate fields in this route
* Do not add project conversion logic yet

Suggested route if needed:
`PATCH /api/estimates/:id/status`

Payload:
`{ status: "draft" | "sent" | "approved" | "declined" }`

Validate status server-side.
