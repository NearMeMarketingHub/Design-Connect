We are starting a Super Admin cleanup in phases. For this first phase, only focus on restructuring the Super Admin area navigation and page layout. Do not change business logic, billing logic, invite logic, or user permissions yet unless required to preserve the current behavior.

The Super Admin dashboard has become one long page with many unrelated sections stacked together. We want the Super Admin side to work more like a proper back-office admin app with a sidebar navigation and separate pages for each major section.

Goal:
Create a dedicated Super Admin layout with left sidebar navigation, and split the current long Super Admin dashboard into separate routed pages.

Use this sidebar structure:

Overview

Customer Management
- Companies
- Demo Requests
- Users
- Invites

Platform Activity
- Projects

Billing & Access
- Pricing & Access

Support & Admin
- Audit Log
- View As User
- Admin Tools

- /admin/dashboard or /admin/overview → Overview
- /admin/companies → Companies
- /admin/demo-requests → Demo Requests
- /admin/users → Users
- /admin/invites → Invites
- /admin/projects → Projects
- /admin/pricing-access → Pricing & Access
- /admin/audit-log → Audit Log
- /admin/view-as-user → View As User
- /admin/tools → Admin Tools
